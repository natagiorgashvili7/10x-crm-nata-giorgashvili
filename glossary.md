
**Authentication** — The process of verifying a user's identity before granting access to protected content.
ავტორიზაცია — მომხმარებლის ვინაობის დადასტურების პროცესი, სანამ მას დაცულ კონტენტზე წვდომას მისცემენ.

**Session** — A temporary object stored while a user is logged in, tracking who they are until they log out.
სესია — დროებითი ობიექტი, რომელიც ინახება მომხმარებლის ავტორიზებული ყოფნისას და აჩვენებს ვინ არის შესული, სანამ არ გამოვა სისტემიდან.

**Validation** — Checking that user input (like a form field) meets required rules before accepting it.
ვალიდაცია — მომხმარებლის მიერ შეყვანილი მონაცემების (მაგ. ფორმის ველების) შემოწმება საჭირო წესებზე, სანამ მათ მიღებენ.

**Fetch** — A built-in JavaScript function used to make network requests, such as getting data from an API.
Fetch — JavaScript-ის ჩაშენებული ფუნქცია, რომელიც გამოიყენება ქსელური მოთხოვნების გასაგზავნად, მაგალითად API-დან მონაცემების მისაღებად.

**Endpoint** — A specific URL that an API exposes for a particular action, like fetching or deleting a resource.
Endpoint — კონკრეტული URL მისამართი, რომელსაც API გვთავაზობს გარკვეული მოქმედებისთვის, მაგალითად რესურსის მიღება ან წაშლა.

**Request method** — The type of HTTP action being performed, such as GET (read), POST (create), or DELETE (remove).
Request method — HTTP მოქმედების ტიპი, რომელიც სრულდება, მაგალითად GET (წაკითხვა), POST (შექმნა) ან DELETE (წაშლა).

**JSON** — A lightweight text format used to structure and exchange data between a browser and a server.
JSON — მსუბუქი ტექსტური ფორმატი, რომელიც გამოიყენება მონაცემების სტრუქტურირებისა და ბრაუზერსა და სერვერს შორის გაცვლისთვის.

**State** — The current data an application is working with at any given moment, such as the list of clients being displayed.
State — მონაცემები, რომლებთანაც აპლიკაცია მუშაობს კონკრეტულ მომენტში, მაგალითად ეკრანზე გამოსახული კლიენტების სია.

**Event listener** — A function that waits for a specific user action (like a click or form submit) and runs code in response.
Event listener — ფუნქცია, რომელიც ელოდება მომხმარებლის კონკრეტულ მოქმედებას (მაგ. დაწკაპუნება ან ფორმის გაგზავნა) და პასუხად უშვებს კოდს.

**Deployment** — The process of publishing a finished project to a live server so others can access it online.
Deployment — დასრულებული პროექტის ცოცხალ სერვერზე გამოქვეყნების პროცესი, რათა სხვებმა ონლაინ შეძლონ მასზე წვდომა.
